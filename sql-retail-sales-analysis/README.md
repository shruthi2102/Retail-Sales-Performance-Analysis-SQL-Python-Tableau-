# Retail Sales Analysis using SQL

## Project Overview
This project analyzes retail sales data using SQL to uncover key business insights related to revenue, customer behavior, product performance, and regional sales.

The project demonstrates practical SQL techniques commonly used by Data Analysts.

---

## Dataset Tables

### retail_sales
Stores transaction-level sales data.

Columns:
- order_id
- order_date
- customer_id
- product
- product_id
- quantity
- price
- city

### customers
Stores customer information.

Columns:
- customer_id
- customer_name
- city
- signup_date

### products
Stores product details.

Columns:
- product_id
- product_name
- category

---

## SQL Techniques Demonstrated

This project covers a wide range of SQL concepts:

- Aggregations (SUM, AVG, COUNT)
- GROUP BY and HAVING
- JOIN operations
- Window functions (RANK, ROW_NUMBER, LAG)
- Running totals
- Subqueries
- Common Table Expressions (CTE)
- Customer lifetime value analysis
- Revenue share calculations
- Time-series analysis

---

## Example Business Questions Answered

- What is the total revenue generated?
- Which products generate the highest revenue?
- Who are the top customers?
- Which cities contribute most to sales?
- What is the monthly revenue trend?
- Which customers place repeat orders?
- What percentage of revenue comes from each product?
